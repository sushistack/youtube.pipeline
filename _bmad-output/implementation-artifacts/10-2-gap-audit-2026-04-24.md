# Story 10.2 Gap Audit

Date: 2026-04-24
Scope: Current worktree snapshot in `/mnt/work/projects/youtube.pipeline`

## Summary

This is not a case of "one missing function."

For Story 10.2, the current branch is missing multiple connected layers required to make the Tuning surface work end-to-end:

1. Runtime Critic evaluator wiring for Golden / Shadow / Fast Feedback
2. `/api/tuning/*` backend surface
3. Run-level prompt version persistence
4. Tuning frontend contracts, hooks, and UI sections
5. Story-specific test coverage

The largest confirmed hole is that `internal/critic/eval` already defines reusable evaluation runners, but the production runtime still lacks the text-provider implementation needed to execute a real Critic evaluation path from the app.

## Confirmed Gaps

### 1. Production text LLM runtime is missing

Impact:
Story 10.2 requires Golden, Shadow, and Fast Feedback to execute the real Critic path from the app. The reusable eval runners exist, but the production text-generation clients that would power a real `eval.Evaluator` are not implemented in this branch.

Evidence:

- `internal/critic/eval/eval.go`
  - Defines `type Evaluator interface { Evaluate(ctx context.Context, fixture Fixture) (VerdictResult, error) }`
  - Comment explicitly says: "the real Critic wires in later"
- `internal/llmclient/deepseek/doc.go`
  - Package doc only
- `internal/llmclient/gemini/doc.go`
  - Package doc only
- `internal/llmclient/`
  - Contains limiter/retry utilities and DashScope TTS only
  - No production text client implementation files were found

Why this matters:

- `internal/critic/eval.RunGolden(...)` and `RunShadow(...)` are reusable
- `internal/pipeline/agents.NewPostWriterCritic(...)` expects a real `domain.TextGenerator`
- The current branch has test fakes for `domain.TextGenerator`, but no production implementation visible in the runtime tree

### 2. No tuning API handler/service layer exists yet

Impact:

Evidence:

  - No `/api/tuning/*` routes
- `internal/api/`
  - No `handler_tuning.go`
- `internal/service/`
  - No tuning-focused service file found

Missing backend surface from the story:

- `POST /api/tuning/fast-feedback`
- `GET /api/tuning/golden`
- `POST /api/tuning/golden/run`

- `POST /api/tuning/shadow/run`
- `GET /api/tuning/calibration`

### 3. Run prompt-version persistence is not present

Impact:
AC-3 requires new runs to persist the active Critic prompt version/hash. That data model is not present in the current branch.

Evidence:

- `internal/domain/types.go`
  - `Run` has no `critic_prompt_version`
  - `Run` has no `critic_prompt_hash`
- `web/src/contracts/runContracts.ts`
  - `runSummarySchema` has no prompt version/hash fields
- `migrations/`
  - Latest visible migration is `012_settings_state.sql`
  - No migration adding prompt-version columns to `runs`

Missing pieces:

- DB migration for `runs.critic_prompt_version`
- DB migration for `runs.critic_prompt_hash`
- `internal/db/run_store.go` scan/select/insert plumbing
- API response serialization updates
- TypeScript schema updates

## Practical Conclusion

The current branch does not show a finished vertical slice for Story 10.2.

Most important gap order:

1. Production text-generator wiring for a real Critic-backed `eval.Evaluator`
2. Tuning backend endpoints and service layer
3. Run prompt-version persistence
4. Tuning frontend contracts/hooks/components
5. Tuning test coverage

If we start implementation now, the safest path is:

1. Confirm whether the missing text-provider runtime exists on another branch or was simply never landed
2. If not, treat evaluator/runtime wiring as a prerequisite subtask of Story 10.2
3. Then build the tuning API and frontend on top of that runtime
