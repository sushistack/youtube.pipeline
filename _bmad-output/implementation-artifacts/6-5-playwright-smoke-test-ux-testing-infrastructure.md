## Story

As a developer,
I want a Playwright smoke test and UX testing infrastructure,
so that the SPA is verified end-to-end and component tests have a solid foundation.

## Prerequisites

**Hard dependency:** Story 6.4 owns the Go-side embedded SPA serving contract that FR52-web depends on. The Playwright smoke test for this story must run against the real Go-served SPA contract, not a Vite-only shortcut path.

**Practical dependencies:** Stories 6.1, 6.2, and 6.3 define the UI shell, route structure, theme tokens, and keyboard semantics this story must verify rather than reinvent.

- Story 6.1 already established the token system and `getComputedStyle()`-based theme contract tests in `web/src/styles/theme-contract.test.tsx`.
- Story 6.2 already created the route shell and `/production`, `/tuning`, `/settings` navigation structure in `web/src/App.tsx` and `web/src/components/`.
- Story 6.3 already scoped the keyboard shortcut invariance rule and global shortcut behavior; Story 6.5 should verify that infrastructure from the outside, not re-specify keyboard semantics.
- The current web project already has Vitest + RTL configured via `web/vitest.config.ts` and `web/src/test/setup.ts`, but it does not yet expose a shared `renderWithProviders.tsx` utility.
- The current `web/package.json` does not yet define a `test` script, does not yet include Playwright, and does not list `@tanstack/react-query` or `zod` as direct dependencies even though Story 6.5 requires `QueryClient` and Zod-based contract parsing. Treat these as expected additions rather than surprises during implementation.
- `testdata/contracts/` already contains checked-in JSON fixtures and schemas. Per FR51 and the architecture, these files are the single source of truth and must never be auto-regenerated by the web test suite.

   - the rule still fails when `Escape`/`Esc` is bound to a non-secondary action

   Rules:
   - Story 6.5 should verify integration of the rule, not re-implement the rule logic if Story 6.3 already owns it
   - if Story 6.3 is not yet merged, this story may need to add the minimum fixture/test wiring needed to prove the lint contract end to end
   - keep failure messages actionable and aligned with the UX invariance intent

   Tests:
   - lint fixtures or rule tests demonstrate pass/fail behavior for compliant and non-compliant bindings
   - `npm run lint` remains part of the story verification path

## Dev Notes

### Story Intent and Scope Boundary

- Story 6.5 is the web-side verification layer for Epic 6, not a new product surface. Its job is to make the existing shell safer to evolve.
- The highest-value outcome is confidence that the SPA shell, route rendering, theme token contract, and keyboard semantics are continuously checkable.
- Keep the scope deliberately small: one Playwright smoke spec, one provider-backed test helper, focused contract parsing, and verification wiring.

### Current Codebase Reality

- `web/src/App.tsx` already redirects `/` to `/production` and renders route shells for Production, Tuning, and Settings.
- `web/src/App.test.tsx` already verifies route rendering and shell behavior in jsdom, which gives Story 6.5 a good baseline for migrating some tests to `renderWithProviders`.
- `web/src/styles/theme-contract.test.tsx` already verifies theme tokens, density variables, and typography contracts; Story 6.5 should extend this coverage to the explicit 16-token non-empty assertion required by the story prompt rather than inventing a separate disconnected style test.
- `web/src/test/setup.ts` already blocks non-localhost fetches and provides a storage shim. Reuse it.
- `testdata/contracts/` already contains the contract fixtures the architecture expects both Go and web tests to validate.

### Architecture Guardrails

- FR51/FR52 treat test infrastructure as a first-class artifact; this story should add durable testing primitives, not throwaway examples.
- The architecture explicitly calls for:
  - `web/playwright.config.ts`
  - exactly one Chromium-only smoke spec
  - `web/src/test/renderWithProviders.tsx`
  - Zod-backed web contract validation against `testdata/contracts/`
- Keep jsdom as the unit/integration environment for Vitest. The architecture explicitly favors jsdom over happy-dom because CSS/style behavior is more trustworthy here.
- Keep contract schema ownership anchored to `testdata/contracts/`; web-side Zod code is a consumer of that truth, not a replacement for it.

### Dependency and Tooling Notes

- `react-router` is already present and supports `MemoryRouter` for tests.
- `@tanstack/react-query` is not currently declared in `web/package.json`; Story 6.5 will likely need to add it to satisfy the `QueryClient` requirement.
- `zod` appears in `web/package-lock.json` transitively, but it is not a direct dependency in `web/package.json`; add it explicitly before relying on it in the test suite.
- Playwright is not currently configured in the `web/` project and will need both package dependency and config/script wiring.
- `web/package.json` currently lacks a `test` script, so the story should standardize the command surface instead of assuming one exists.

### Implementation Guidance

- Prefer colocating new test-only helpers under `web/src/test/` so the web testing surface stays discoverable.
- For `renderWithProviders`, return RTL's render result plus any router/query helpers only if they are actually used; keep the helper small.
- For Playwright keyboard assertions, target stable shell affordances that already exist or are guaranteed by the shell stories. Avoid brittle selectors tied to placeholder prose.
- If Story 6.4's Go-serve command is inconvenient for Playwright bootstrapping, create a thin scripted start path rather than bypassing the production-like server contract entirely.
- Keep the smoke spec deterministic and CI-friendly. One stable spec is more valuable than several flaky ones.

### Testing Requirements

- Expected verification path for story completion:
  - `npm run lint`
  - web Vitest suite for unit/component/style/contract tests
  - Playwright Chromium smoke test
- The smoke test must verify:
  - SPA load succeeds
  - `/production` content renders
  - `Enter` and `Escape` produce the expected interaction on a stable, keyboard-focusable surface
- Contract tests must read checked-in fixtures and fail on schema drift.
- Style tests must assert computed custom property values, not just raw CSS source text.

### Previous Story Intelligence

- Story 6.1 already put the design-token system under test, so Story 6.5 should build on that rather than duplicating the entire token suite.
- Story 6.2 already established the route shell contract and `/production` route, which gives this story a stable smoke-test target.
- Story 6.3 already defined keyboard shortcut semantics and lint invariance. Story 6.5 is the point where those guardrails should be exercised as part of the routine web verification path.
- Story 6.4 established the production serving contract that Playwright should trust. This story should avoid drifting back to a test setup that only works against a disconnected dev-only shell.
