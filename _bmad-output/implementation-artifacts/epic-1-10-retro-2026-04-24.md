## 2. Successes & Strengths (by epic)

Synthesized from story files' Dev Notes / Implementation Notes / Breakthroughs sections.

**Epic 1.** Three-layer API isolation (constructor injection → `BlockExternalHTTP` runtime guard → CI secret lockout) locked in Day 1 — Stories 2+ never faced external-call leakage. Layer-import linter enforced module boundaries from the first commit; caught 4 purity violations throughout Epics 3, 6, and 8. Go `embed.FS` for migrations at project root (not nested under `internal/`) avoided circular-path pain downstream.

**Epic 2.** `NextStage(current, event) → (Stage, error)` pure-function state machine with `IsHITLStage()` / `StatusForStage()` proved reusable — every subsequent epic inherited the enum+validator pattern. `FakeClock` injection for timing assertions eliminated flaky tests across the whole codebase. HITL wait points explicitly enumerated prevented off-by-one errors in cost tracking (Story 2.4) and pause-resume (Story 2.6).

**Epic 3.** Named-field AgentFunc struct (vs `[]AgentFunc` slice) forced compiler-enforced completeness when a new agent was added. Atomic write pattern (temp file → `Sync` → `os.Rename`) inherited from Story 2.3 prevented partial-write races on `scenario.json`. Spy-agent inline closures (no mocking libraries) gave deterministic chain-order verification.

**Epic 4.** Shadow-mode eval runner — parallel execution without blocking live pipeline. Immutable `testdata/contracts/` manual-approval rule prevented silent schema drift. Cohen's kappa as a pure statistical function — no corpus dependency at the computation layer.

**Epic 5.** Non-cancelling `errgroup.Group` (NOT `WithContext`) — image track failure doesn't cancel TTS, enabling operator resume choices. `CallLimiter` interface allowed Stories 5.2–5.5 to inject fakes without `httptest.Server`. Clean-slate Phase B resume (DELETE segments + `ClearImageArtifacts` + `ClearTTSArtifacts`) rebuilds deterministically.

**Epic 6.** Tailwind CSS 4 + CSS variables (dark-only) avoided shadcn theme complexity. Go `embed.FS` serving React `dist/` at build time — zero runtime asset pipeline. `useKeyboardShortcuts` hook with `allow_in_editable` centralized what would have been ad-hoc event handling.

**Epic 7.** Master-detail layout with Zustand selection state + React Router dynamic segments enabled scene-level navigation without page reloads. `ProductionShortcutPanel` scoped hook registration prevented J/K conflicts with `BatchReview` (Epic 8).

**Epic 8.** `superseded_by` soft-delete on decisions preserved audit trail while enabling undo. `context_snapshot` JSON payload flexibility — avoided schema migration when `CommandKind` enum expanded. Decision store's existing `superseded_by IS NOT NULL` filter meant counts/diffs worked transparently after undo.

**Epic 9.** `ffmpeg-go` fluent builder prevented shell-escaping bugs in complex `filter_complex` strings. Artifact atomicity pattern reused from Phase A. Pre-upload compliance checklist on both frontend and backend — defense in depth.

**Epic 10.** Golden/Shadow CI gate with Critic-file path filter — fixture-expectation evaluator runs without LLM keys, safe in the no-secrets CI posture. One-click data export consolidates scattered artifact locations into a single JSON/CSV dump with CSV-injection guard.

**Cross-epic wins.** Atomic write pattern reused everywhere (scenario.json, metadata.json, manifest.json). Pure functions + injectable deps + `FakeClock` → zero flaky tests in the 700+ Go test suite. `ncruces/go-sqlite3` (pure Go, no CGO) kept CI simple even without `-race`.

## 3. Challenges & Growth Areas (by epic)

**Epic 1.** Vite 8.x auto-installed by shadcn but explicitly banned — required immediate downgrade to 7.3 before anything else could proceed. `LoadRunStateFixture` initially omitted PRAGMA alignment with production `OpenDB` — caught in review. WAL sidecar files (`-wal`, `-shm`) inherit default umask permissions, not 0600; accepted for localhost.

**Epic 2.** `Engine.Advance` left as explicit stub (`"advance not implemented: epic 3 scope"`) — preserved through Story 3.1 with guard test. **Never wired since.** Anti-progress detector bag-of-words `Tokenize` is semantically weak for Korean (primary content language) — will over-trigger FPs on Korean Writer outputs; V1.5 embedding migration is the fix but not scheduled. Migration ordinal collision: `004_anti_progress_index.sql` and `004_hitl_sessions.sql` both exist — tracked in `deferred-work.md` but unresolved.

**Epic 3.** **Researcher & Structurer agents are V1 deterministic non-LLM placeholders** — they read `/mnt/data/raw` corpus and produce outputs by field-copy + modulo distribution. Prompt templates at `docs/prompts/scenario/01_research.md`, `02_structure.md` exist but are **guarded by `TestAgents_NoPromptFileReferences` to stay unreferenced**. Only Writer + Critic path real LLM calls. Real LLM quality for Phase A research/structure is unmeasured.

**Epic 4.** Real golden-set data population (100+ decisions for kappa calibration) deferred to post-V1 when pipeline actually produces artifacts. `LoadShadowInput` cannot resolve `scenario_path="scenario.json"` against live DB path `{outputDir}/{runID}` — fixtures sidestep the issue; production Shadow against a real run fails every case. Surfaces when Story 10.4 CI wiring hits live DB.

**Epic 5.** Rate-limiter `fn` goroutine may outlive `Do` on timeout if callee ignores `ctx` — can silently exceed `MaxConcurrent`. Character candidate resolved once per run; mid-run operator re-selection not detected. Several test doubles are tautological — canonical-path assertions verify "we sent the right thing" not "the thing landed", and the cancellation sibling fake ignores `ctx.Done()`, making the non-cancellation assertion vacuous.

**Epic 6.** `internal/api/spa.go` serves `index.html` with HTTP 200 for misses under `/assets/*` — stale-build regression masks CSS/JS 404s. `warnings` field nesting drift between Go handler ↔ contract fixture ↔ Zod schema (documented but unfixed). `renderWithProviders` gives fresh `QueryClient` but leaves `useUIStore` zustand singleton shared across tests. **Root `e2e/smoke.spec.ts` is `test.todo('FR52-web smoke: SPA loads and /production renders')`** — canonical web smoke is a placeholder; real tests live in `web/e2e/` but CI doesn't run them.

**Epic 7.** TOCTOU in `SceneService.EditNarration` (check + write non-atomic); backend Edit endpoint echoes request payload not DB readback. `image_track.go` override swaps descriptor prefix only — individual shot `VisualDescriptor` strings composed in Phase A may still embed pre-edit descriptor text. Several dead-code exports (`SetFrozenDescriptor`, `BindFlags`) ship with no callers.

**Epic 8.** **`RetryExhausted` `>=` vs `>` mismatch across three sites** (`ListReviewItems` / `RecordSceneDecision` / `DispatchSceneRegeneration`). `BatchApprove` undo trusts caller-supplied `aggregate_command_id` in `context_snapshot` without server normalization — cross-run undo blast-radius risk. `approved_scene_indices` snapshot duplicated per-decision-row — O(N²) storage. Undo stack never GC'd on run switch. Ctrl+Z only — macOS Cmd+Z unmapped. `/regen` has no idempotency key. `/api/runs/{id}/status` response has no Zod counterpart.

**Epic 9.** `xfade offset = DurationSeconds - 0.5` can go negative for shots < 0.5s — FFmpeg undefined behavior. `probeDuration` returns 0 silently passes `outputDur > 0.1` check — partial-failure clip fake-passes. `metadata.json + manifest.json` pair is **non-atomic**: first write succeeds, second fails → timestamp-mismatched retry. Single-quote in file path breaks FFmpeg concat list (runDir system-generated, low risk but undefended). Duration tolerance accumulates codec-level rounding >0.1s for >20-clip videos.

**Epic 10.** **Story 10-2 tuning surface is unbuilt.** The `10-2-gap-audit-2026-04-24.md` enumerates five missing connected layers: (a) production text LLM runtime for real Critic wiring (`deepseek/doc.go`, `gemini/doc.go` are docs only), (b) `/api/tuning/*` backend surface (handler + service), (c) run-level prompt-version persistence (migration 013 added since audit, but not fully wired), (d) Tuning frontend contracts/hooks/UI (placeholder shell), (e) story-specific test coverage. `RunGolden` mutates `testdata/golden/eval/manifest.json` on every invocation (dev-box dirty-tree). `go-version: '1.25.7'` in CI doesn't exist as a Go release.

## 4. Key Insights & Lessons Learned

**4.1. Per-phase integration rich, cross-phase seam barren.** Integration tests exist for Phase A chain, cost caps, resume, observability, anti-progress, HITL service, Shadow eval — but **no test composes Phase A → B → C on a seed SCP**. The most load-bearing seam has zero coverage. This is the single highest-impact gap in the codebase.

**4.2. CI signal integrity erosion.** `test-e2e` has `continue-on-error: true` (line 46 of ci.yml) with a stale comment (`"Remove once serve command exists"` — Epic 6 shipped serve). Root `e2e/smoke.spec.ts` is `test.todo()`. Phase C tests skip when ffmpeg is missing; `test-go` job doesn't install ffmpeg. Three independent mechanisms combine into **green-but-unguarded**.

**4.3. Test-double audit overdue.** Multiple deferred items (5-4, 5-5, 7-1, 6-5 areas) note that fake providers honor inputs they receive, so tests assert shape-of-dispatch not effect-of-execution. No project-wide audit of test-double fidelity has been run.

**4.4. Migration ordinal discipline slipped.** `004_anti_progress_index.sql` and `004_hitl_sessions.sql` both exist. The runner is lexicographic per run but the shared ordinal is semantically broken. No migration-ordinal lint test exists. This is the first silent convention violation of the project.

**4.5. "Scope leakage" is the dominant review theme.** `deferred-work.md` contains ~48 entries that explicitly attribute work to the wrong story (e.g., "2-7 concern leaked into 2-6's file", "Story 8.1 scope caught in 8.6 review"). The project's commit-scope discipline (Jay's feedback memory: no out-of-scope edits in restricted commits) caught most of these, but the story-file / review-file separation often lags. **This is a process issue, not a code issue.**

**4.6. V1 placeholder architecture is well-marked but under-tracked.** Researcher/Structurer non-LLM, `Engine.Advance` stub, root `e2e/smoke.spec.ts` todo, `10-2` unbuilt, `LoadShadowInput` production path broken — all are **documented where they live** (in story files, in `t.Skip` messages, in code comments). But **none are cross-indexed** in a single place that answers "what does V1 actually not do yet?". This retrospective is the first such index.

**4.7. Commit history pattern: "story X" → "story X code review fixes" → sometimes "story X second code review pass".** Almost every story has 2–3 commits. Code review catches real issues every time. This is working as intended — the "second code review pass" on 10-5 (CSV injection) specifically proved the value of re-reviewing post-initial-fix. No change recommended.

6. Phase C tests skip without ffmpeg; CI `test-go` doesn't install ffmpeg.
7. No cross-phase integration test — only per-phase.

### Risk call: HIGH

(b) Replace root `e2e/smoke.spec.ts` with `web/e2e/` tests OR repoint CI. Remove `continue-on-error`.
